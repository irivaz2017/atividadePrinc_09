WorldWideWeb - Resumo

O projeto WWW ombina as técnicas de recuperação de informação e hipertexto para criar um sistema de informação global fácil, mas poderoso.

O projeto é baseado na filosofia de que muita informação acadêmica deve estar disponível gratuitamente para qualquer pessoa. Destina-se a permitir a partilha de informação dentro de equipas dispersas internacionalmente, e a divulgação de informação por grupos de apoio. Originalmente voltado para a comunidade de Física de Altas Energias, ele se espalhou para outras áreas e atraiu muito interesse em áreas de suporte ao usuário, descoberta de recursos e trabalho colaborativo.

Visualização do leitor

O mundo WWW consiste em documentos e links. Índices são documentos especiais que, em vez de serem lidos, podem ser pesquisados. O resultado dessa pesquisa é outro documento ("virtual") contendo links para os documentos encontrados. Um protocolo simples(" HTTP ") é usado para permitir que um programa de navegador solicite uma pesquisa de palavra-chave por um servidor de informações remoto.

A web contém documentos em vários formatos. Aqueles documentos que são hipertexto, (reais ou virtuais) contêm links para outros documentos, ou lugares dentro de documentos. Todos os documentos, sejam reais, virtuais ou índices, parecem semelhantes ao leitor e estão contidos no mesmo esquema de endereçamento.

Para seguir um link, o leitor clica com o mouse (ou digita um número se não tiver mouse). Para pesquisar e indexar, um leitor fornece palavras-chave (ou outros critérios de pesquisa). Essas são as únicas operações necessárias para acessar todo o mundo dos dados.

Visão do provedor de informações

Os navegadores WWW podem acessar muitos sistemas de dados existentes via protocolos existentes (FTP, NNTP) ou via HTTP e um gateway. Dessa forma, a massa crítica de dados é rapidamente ultrapassada, e o uso crescente do sistema por leitores e fornecedores de informações incentiva uns aos outros.

Fazer uma web é tão simples quanto escrever alguns arquivos SGML que apontam para seus dados existentes. Torná-lo público envolve executar o daemon FTP or HTTP daemon e criar pelo menos um link em sua web a partir de outro. De fato, qualquer arquivo disponível por FTP anônimo pode ser imediatamente vinculado a uma web. O esforço inicial muito pequeno é projetado para permitir pequenas contribuições. No outro extremo da escala, grandes provedores de informações podem fornecer um servidor HTTP com texto completo ou indexação de palavras-chave. Isso pode permitir o acesso a um grande banco de dados existente sem alterar a maneira como esse banco de dados é gerenciado. Esses gateways já foram transformados nos sistemas VMS/Help da Digital, na Technical University of Graz's "Hyper-G" e nos sistemas "WAIS" da Thinking Machine.

O modelo WWW supera as frustrantes incompatibilidades de formato de dados entre fornecedores e leitor, permitindo a negociação de formato entre um navegador inteligente e um servidor inteligente. Isso deve fornecer uma base para extensão em multimídia e permitir que aqueles que compartilham padrões de aplicativos façam uso completo deles na web.

Este resumo não descreve as muitas possibilidades interessantes abertas pelo projeto WWW, como o cache de documentos eficiente. a redução de cópias desatualizadas redundantes e o uso de daemons de conhecimento. Há mais informações na documentação do projeto online, incluindo algumas informações sobre hipertexto e muitas notas técnicas.
